# Инструкция по деплою на Beget (Node.js)

Этот проект состоит из двух частей: **Frontend** (React + Vite) и **Backend** (Node.js + Express). На Beget мы будем запускать Node.js сервер, который будет раздавать и API, и статику фронтенда.

## 1. Подготовка в панели Beget

1. Зайдите в раздел **"Сайты"** -> **"Создать новый сайт"**.
2. В настройках сайта выберите **Node.js**.
3. Рекомендуемая версия Node.js: **20 и выше**.
4. Укажите домен (или поддомен), на котором будет работать сайт.

## 2. Загрузка кода

### Вариант А: Через Git (рекомендуется)
Если вы настроили Git в панели Beget:
1. В разделе Node.js выберите **"Установка из Git"**.
2. Укажите ссылку на ваш репозиторий.

### Вариант Б: Через FTP/Менеджер файлов
1. Загрузите все файлы проекта в корневую папку сайта.
2. **ВАЖНО:** Не загружайте папку `node_modules` (она слишком тяжелая и должна быть установлена на сервере).

## 3. Настройка и установка зависимостей

Зайдите на сервер через **Terminal** в панели Beget или по SSH:

```bash
# Перейдите в папку проекта
cd ~/имя_вашего_сайта/public_html

# Установите зависимости для фронтенда и соберите его
npm install
npm run build

# Установите зависимости для сервера
cd server
npm install
```

## 4. Переменные окружения (.env)

В папке `server` создайте файл `.env` (или добавьте переменные в панели Beget -> Node.js -> Переменные окружения):

```env
PORT=3001
JWT_SECRET=ваш_очень_секретный_код_2024
CORS_ORIGIN=https://ваш-домен.ru
```

## 5. Запуск приложения

В панели Beget в настройках Node.js приложения укажите:
- **Точка входа (Startup file):** `server/index.js`
- **Папка приложения:** Корневая папка проекта.

После этого нажмите **"Запустить"** или **"Перезапустить"**.

---

## Что делать, если на Beget нет Node.js (обычный хостинг)?

Если у вас тариф без поддержки Node.js, вы можете использовать только фронтенд (статику), но админка работать не будет. В этом случае:
1. Соберите проект локально: `npm run build`.
2. Загрузите содержимое папки `dist` в `public_html`.

Но для работы редактирования цен **Node.js обязателен**.

## Резервное копирование данных
Все данные сайта (цены, услуги, отзывы) хранятся в файле `server/data.json`. 
**Рекомендуется скачивать этот файл себе на компьютер раз в неделю**, чтобы у вас всегда была резервная копия настроек сайта. 
Если вы захотите переехать на другую базу данных (MySQL), я подготовил файл `server/database.sql` с готовой структурой таблиц.
